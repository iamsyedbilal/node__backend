# Node.js Backend Journey - Day 1

Welcome to the Node.js Backend Journey! This repository documents the day-by-day progress of learning and building backend applications using Node.js. Each folder represents a specific day or topic covered during the journey.

## Day 1: Events and Chat App

- **events.js**: Demonstrates the use of Node.js events.
- **chatApp/**: Contains a simple chat application.
  - `chat.js`: Chat logic.
  - `index.js`: Entry point for the chat application.

## Day 2: HTTP Server

- **http_server/**: Basic HTTP server implementation.
  - `index.js`: HTTP server logic.
  - `package.json`: Dependencies for the HTTP server.
- **http_server_1/**: Another HTTP server example.
  - `index.js`: HTTP server logic.
  - `log.txt`: Log file for the server.
  - `package.json`: Dependencies for the HTTP server.

## Day 3: Handling File

- **handlingFile/**: File handling examples.
  - `todo.js`: Script for managing a TODO list.
  - `todo.json`: JSON file storing TODO data.

## Day 4: Node.js with Nginx

- **node\_\_nginx/**: Node.js with Nginx examples.
  - `about.html`, `contact.html`, `index.html`: HTML files for the Nginx server.
  - `index.js`: Node.js logic for the Nginx server.

## Day 5: Express JS

- **express_basics/**: Introduction to Express.js.
  - `index.js`: Basic Express.js server setup.
  - `routes.js`: Example of routing in Express.js.
- **express_middleware/**: Middleware in Express.js.
  - `index.js`: Demonstrates the use of middleware.
  - `logger.js`: Custom middleware for logging requests.
- **express_templates/**: Templating with Express.js.
  - `index.js`: Example of using a templating engine.
  - `views/`: Directory containing template files.

### How to Run

1. Navigate to the desired folder.

### Notes

- Ensure Node.js is installed on your system.
- Use `npm install` in folders with a `package.json` file to install dependencies.
